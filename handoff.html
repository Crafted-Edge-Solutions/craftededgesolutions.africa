Wellness Solutions Africa
redesign brief.

Everything needed to bring wellness-solutions.africa to the same craft level as craftededgesolutions.africa — adapted to wellness aesthetics, implemented in Django + Wagtail. Pair this document with the design playbook and the visual concept.

— StackDjango · Wagtail · PostgreSQL
— Timeline4–6 weeks (3 phases)
— ReferenceCrafted Edge site + playbook
— OwnerCrafted Edge Studio
— 00 / Brief

Why we're
doing this.

Wellness Solutions Africa is operationally strong but visually under-positioned. The current site reads like a generic wellness directory — when the practice is, in fact, a senior, slow-medicine clinic with serious clinical depth. The redesign is a positioning move first, a visual one second.

— Goals

  • Reposition from "wellness centre" to "considered medical practice"
  • Reflect the slowness, sensory care, and senior practitioner team
  • Convert qualified clients via clearer booking and pre-consultation flows
  • Unify across services — currently each modality reads as its own brand
  • Establish a design system the in-house team can extend without dilution

— Non-goals

  • Copying craftededgesolutions.africa — the DNA transfers, not the surface
  • Adding emoji, illustrated icons, or generic wellness imagery
  • Building a "wellness commerce" experience (no product shop in v1)
  • Migrating off Wagtail or away from Django
— 01 / Scope

Three phases.
Shipping incrementally.

Phase 01 · Foundation

Weeks 1–2

Lock the design tokens, type system, and reusable Wagtail blocks. No public-facing pages migrate yet — but everything needed to build them does. Engineering can start touching production code.

  • Design tokens in static/css/tokens.css
  • Base templates with sticky nav, footer, page shell
  • StreamField block library (hero, numbered list, stat grid, practitioner card, CTA band)
  • Motion layer (static/js/motion.js) wired globally

Phase 02 · Pages

Weeks 3–4

Migrate the core public pages onto the new system. Pages built from blocks, content edited via Wagtail admin. Live preview enabled; non-public until full sign-off.

  • Home · Approach · Practitioners · Offerings · Contact
  • Booking flow re-styled (existing logic preserved)
  • Image content audit — replace stock imagery with editorial
  • SEO + analytics parity with current site

Phase 03 · Polish & launch

Weeks 5–6

Final motion tuning, image optimisation, performance pass, and content review. DNS swap and launch. Two weeks of post-launch monitoring included.

  • Performance: Lighthouse ≥ 95 across all metrics
  • WCAG AA accessibility audit
  • Cross-browser testing (Chromium, WebKit, Firefox)
  • Editorial training session with practice manager

— Where each colour goes

  • --accent (sage): primary emphasis — italic display text, status dots, hover states, active nav links, primary button hover. Use 4–6 times per page max.
  • --accent-warm (terracotta): secondary emphasis — error states, calendar 'today' marker, occasional pull-quote. Use 1–2 times per page max.
  • --fg (charcoal): all primary headings and body. Never pure black.
  • --bg-card: raised surfaces — practitioner cards, lifted CTAs, the form wrap. Use sparingly; lines are still preferred.

— Italic emphasis pattern

Every display heading on the site has one italic word or phrase in --accent. This is the rhythm of the type system. Examples:

  • "Care that's actually felt."
  • "Slow, considered, specific."
  • "Six disciplines. One philosophy."

Always one italic moment per heading — never two, never zero on a hero heading.

— 04 / Foundation

Spacing & rhythm.

Generous space is part of the brand. Practice this rule: when unsure, add more space, not less.

— Vertical rhythm

  • Section padding: 120px top/bottom on desktop, 80px on mobile
  • Within sections: 56px between section head and content body
  • Between paragraphs: 20px
  • Between heading + first paragraph: 24px
  • Card internal padding: 24–32px depending on density

— Horizontal

  • Container max-width: 1240px
  • Edge gutter: 32px desktop, 22px mobile
  • Grid gap (cards): 28px
  • Grid gap (text columns): 80px
— 05 / Components

Component catalog.

The component library is intentionally small. Twelve pieces compose the entire site. Build them once, well, and reuse everywhere.

01 · Nav (sticky)

Global

Fixed top bar. Logo mark + sage circle on the left, mono nav links centred-right, primary CTA on the right. Tightens on scroll. Backdrop blur over the cream background.

02 · Hero block

Page template

Two-column on desktop, stacked mobile. Left: eyebrow → display heading with italic moment → lead paragraph. Right: editorial photograph (3:4 aspect ratio). Meta strip beneath shows founded/location/practitioner count.

03 · Numbered row list

StreamField

Three-column grid: number, heading, body. Hover state shifts content right and reveals arrow. Used for offerings, principles, process steps. Replaces every icon-and-card layout in the current site.

04 · Practitioner card

StreamField

Vertical card: 4:5 portrait photo on top, name (with italic surname accent) + mono role label + 1-sentence bio. Hover lifts the card 4px. Group in 3-up grid.

05 · Stat grid

StreamField

Four cells in a horizontal row separated by 1px lines. Each cell: large display number (with italic suffix), mono label, one-sentence description. Numbers animate from 0 on scroll into view.

06 · Pull section (philosophy / approach)

StreamField

Two-column: 1fr / 2fr. Left: eyebrow + display heading. Right: 2–4 paragraphs of body. Used for "Our approach", "Why we built this", and editorial copy moments.

07 · CTA band

Page template

Two-column footer-band: big display headline left, contact details + buttons right. Each contact row separated by 1px line with hover underline animation. Appears at the bottom of most pages.

08 · Form wrap

Booking + contact

Single raised card (--bg-card), 18px radius, 40px padding. Mono labels above each field. Inputs transparent with 1px line under. Focus state changes border to --accent. Primary submit at full width.

09 · Booking calendar

Existing logic

Re-style only — preserve booking backend logic. Wrap in form-wrap container. Selected date in --accent, today's date in --accent-warm. Time slots as mono pills.

10 · Footer

Global

Four columns: brand block (logo + tagline + contact), studio/practice links, offerings links, contact info. Bottom strip with © and "By appointment only · Westlands, Nairobi".

11 · Image block

StreamField

Single editorial image, full-bleed within container, with optional caption in mono. Constrained to specific aspect ratios (3:4 portrait, 16:9 landscape, 1:1 square). Lazy-loaded.

12 · Quote / testimonial

StreamField

Single quote on its own band. Display-size italic text in --fg (not accent — quotes should feel grounded), 60ch max-width, centred. Attribution in mono caps beneath.

— 06 / Components

Wagtail blocks.

Build each component above as a typed StreamField block. No RichTextBlock with full HTML access. Every block has a specific template; editors choose content, never markup.

# wellness/blocks.py
from wagtail import blocks
from wagtail.images.blocks import ImageChooserBlock


class HeroBlock(blocks.StructBlock):
    eyebrow = blocks.CharBlock(required=False, max_length=80,
        help_text="Mono label above heading. Lowercase, em-dash prefix added automatically.")
    heading = blocks.CharBlock(max_length=120,
        help_text="Use ALL CAPS for emphasis on hero. Italic word marked with *asterisks*.")
    lead = blocks.TextBlock(max_length=400)
    image = ImageChooserBlock(required=False,
        help_text="3:4 ratio. Editorial portrait, soft natural light, single subject.")

    class Meta:
        template = "blocks/hero.html"
        icon = "openquote"
        label = "Hero"


class NumberedRowBlock(blocks.StructBlock):
    heading = blocks.CharBlock(max_length=60,
        help_text="Use *asterisks* around the italic word.")
    body = blocks.TextBlock(max_length=400)


class NumberedListBlock(blocks.StructBlock):
    eyebrow = blocks.CharBlock(required=False, max_length=80)
    heading = blocks.CharBlock(max_length=120)
    lead = blocks.TextBlock(required=False, max_length=300)
    rows = blocks.ListBlock(NumberedRowBlock(),
        help_text="Numbered automatically — 01, 02, 03...")

    class Meta:
        template = "blocks/numbered_list.html"
        icon = "list-ol"


class PractitionerCardBlock(blocks.StructBlock):
    first_name = blocks.CharBlock(max_length=40)
    surname = blocks.CharBlock(max_length=40,
        help_text="Will appear in italic accent colour.")
    role = blocks.CharBlock(max_length=80,
        help_text="Mono label. Use · between role parts.")
    bio = blocks.CharBlock(max_length=180,
        help_text="One short sentence. Strict 180 char limit.")
    portrait = ImageChooserBlock(
        help_text="4:5 ratio. Natural light, single subject, neutral background.")


class StatCellBlock(blocks.StructBlock):
    value = blocks.CharBlock(max_length=8,
        help_text="Numeric prefix, e.g. '90'. Italic suffix below.")
    suffix = blocks.CharBlock(required=False, max_length=8,
        help_text="e.g. 'min', '+', '/7' — italic in accent colour.")
    label = blocks.CharBlock(max_length=40,
        help_text="Mono label below number.")
    description = blocks.CharBlock(required=False, max_length=80)


class StatGridBlock(blocks.StructBlock):
    eyebrow = blocks.CharBlock(required=False, max_length=80)
    heading = blocks.CharBlock(required=False, max_length=120)
    cells = blocks.ListBlock(StatCellBlock(), min_num=2, max_num=4)

    class Meta:
        template = "blocks/stat_grid.html"


class CTABandBlock(blocks.StructBlock):
    heading = blocks.CharBlock(max_length=120)
    email = blocks.EmailBlock(default="hello@wellness-solutions.africa")
    phone = blocks.CharBlock(default="+254 7XX XXX XXX")
    primary_cta_label = blocks.CharBlock(default="Book a consultation")
    primary_cta_url = blocks.URLBlock(required=False)

    class Meta:
        template = "blocks/cta_band.html"


# Master StreamField — what every page can use
class WellnessStreamBlock(blocks.StreamBlock):
    hero = HeroBlock()
    numbered_list = NumberedListBlock()
    pull_section = PullSectionBlock()
    practitioner_grid = PractitionerGridBlock()
    stat_grid = StatGridBlock()
    image = ImageBlock()
    quote = QuoteBlock()
    cta_band = CTABandBlock()

— Why typed blocks matter

The current site likely uses RichTextBlock liberally. That's how every wellness site looks the same — editors paste raw HTML, fonts shift, headings drift, the design dies in three months.

Typed blocks make the design system uneditable. Editors can change words and images, never markup. The site looks pristine six months after launch because it has to.

— Italic emphasis helper

Editors mark italic phrases with *asterisks*. Template filter renders them with the accent colour.

from django import template
from django.utils.safestring import mark_safe
import re

register = template.Library()


@register.filter
def italic_emphasis(value):
    """Wraps *text* in <em> with accent colour."""
    return mark_safe(
        re.sub(
            r'\*([^\*]+)\*',
            r'<em class="accent">\1</em>',
            value
        )
    )

Used in templates: {{ block.value.heading|italic_emphasis }}

— 09 / Pages

Imagery direction.

Photography is the brand here. Bad images will sabotage everything. We strongly recommend a half-day editorial shoot before launch.

— What to shoot

  • Practitioner portraits (3 minimum, in clinic space, natural light). 4:5 ratio.
  • Treatment room interiors — empty, calm, no people. Highlights wood, light, plants. 3:4 and 16:9 ratios.
  • Detail shots — hands working, herbs, a single open notebook, plant against light. 1:1 square.
  • One hero portrait — practitioner with client (or alone) in consultation. 3:4 ratio. The home page hero.

— What NOT to use

  • Stock photography of yoga poses, smiling families, or generic "wellness"
  • Heavy filters or oversaturated colour grading
  • Multiple people in studio-lit "team photo" setups
  • Equipment-focused product shots

— Technical specs

  • Use Wagtail's {% image %} tag with format-webp + format-avif fallback
  • Required focal point set in admin for every image
  • Lazy load below the fold (loading="lazy")
  • Maximum upload 5MB; Wagtail handles responsive variants
— 10 / Delivery

Motion layer.

The motion library is portable from the Crafted Edge site. Drop it in, retune for the wellness pace.

— What transfers as-is

  • Lenis smooth scroll — same library, same config
  • Scroll-reveal Intersection Observer — fade-up on enter
  • Stat counter — number animates from 0
  • Nav scroll state — tightens on scroll
  • Magnetic primary buttons — subtle cursor pull

— What to retune

  • Easing: use cubic-bezier(0.32, 0.72, 0, 1) instead of (0.22, 1, 0.36, 1) — slower start, gentler finish
  • Duration: 1.4s reveals instead of 1s. Wellness should feel like exhale.
  • Cursor: change ring colour to --accent, increase ring size to 44px
  • Skip on: booking flow, form interactions (always feels snappy)

The full motion.js from this project is the starting point. Wellness team adjusts the easing constants and ships it.

— 11 / Delivery

Performance budget.

— Lighthouse targets (production)

  • Performance: ≥ 95 on home + practitioners + booking
  • Accessibility: ≥ 95 site-wide
  • Best practices: ≥ 95 site-wide
  • SEO: 100 across all public pages

— Specific budgets

  • LCP: < 2.0s on 4G mobile
  • CLS: < 0.05
  • JS bundle: < 80kb gzipped (lenis + motion.js + Wagtail boot)
  • CSS: < 30kb gzipped (single file, no purge needed at this scale)
  • Hero image: < 120kb webp at 1.5x density

— Implementation notes

  • Inline critical CSS in <head>
  • Self-host Fraunces + DM Sans (Google Fonts adds latency)
  • Cloudflare in front of Django for static asset caching
  • WhiteNoise or S3 for media in production
— 12 / Delivery

Acceptance criteria.

Before launch, each item below must be verified. Sign-off requires all checked.

— Design fidelity

Token system locked. Tokens.css in production; no hardcoded colours anywhere.
Typography parity. Fraunces, DM Sans, JetBrains Mono self-hosted with proper fallbacks.
Italic emphasis pattern applied. Every hero heading has one accent-italic phrase.
No icons or emoji on the public site. Numbers replace icons.
Borders over cards. No drop-shadow card patterns; lines define grids.

— Functional

Booking flow preserved. All existing payment + calendar logic working unchanged.
Forms submit successfully. Contact form lands in practice manager inbox; autoresponder sent.
Wagtail admin documented. Practice manager can edit every page block independently.
SEO not regressed. All current URLs return 200 or 301; meta tags + schema in place.

— Technical

Lighthouse ≥ 95 on home, practitioners, booking pages.
WCAG AA verified with axe-core. All interactive elements keyboard-accessible.
Cross-browser tested. Chromium, WebKit, Firefox; iOS Safari + Android Chrome.
prefers-reduced-motion honoured. All motion disables cleanly when set.

— Brand

Imagery shot or sourced editorially. No stock wellness photography.
Copy reviewed for positive framing. No "we don't / we're not" phrasing.
Accent used sparingly. Maximum 6 accent instances per page audit.
Practitioner content approved by each practitioner before going live.

Sign-off: Practice owner + Crafted Edge lead engineer + practice manager (admin training complete). All three sign before DNS swap.
