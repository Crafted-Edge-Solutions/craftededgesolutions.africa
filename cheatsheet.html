Quick Decision: What to Use

Need it in 2 seconds, good quality?   →  groq "..."         (Llama 70B, free)
Private data / offline / instant?     →  ask "..."          (local 3B GPU)
Best local quality, still private?    →  gemma (interactive) (4B GPU)
Coding session with file access?      →  p  or  pask "..."  (Pi + Groq)
Spotify / scheduling / web search?    →  h  or  hask "..."  (Hermes + Groq)
Browse & chat with history?           →  localhost:3000      (OpenWebUI)

groq — Your Workhorse (Llama 3.3 70B, 2 sec, free)

# Direct prompts
groq "write a blog post about X"
groq "proposal email to [client] for [project]"
groq "5 LinkedIn posts about web design for Kenyan SMEs"
groq "reply to this client complaint professionally: [paste]"

# Pipe content in (for analysis / review)
cat file.txt | groq "summarise this"
head -50 index.html | groq-review "review meta tags for SEO"
cat contract.txt | groq "what are the key risks in this?"

# Save directly to file
groq "write SEO post about mobile websites in Kenya" > ~/content/draft.md
groq "write 10 email subject lines for a website launch offer" >> ~/content/subjects.txt

# Fast model (lighter tasks, even quicker)
groq-fast "give me 3 tagline options for a web agency"

Limits: 14,400 req/day · ~30 req/min · No system prompt overhead

ask / askbig — Local GPU (Private, No Internet)

ask "write a WhatsApp follow-up for a client who went quiet"
ask "translate this to Swahili: We'd love to work with you"
askcode "write a JavaScript contact form validator"
askbig "explain this contract clause: [paste]"    # gemma3 4B, better quality

# Interactive sessions
qwen       # qwen2.5 3B — general chat, 30 tok/s
gemma      # gemma3 4B — best local quality, ~15 tok/s
coder      # qwen-coder 3B — code, 30 tok/s
think      # deepseek-r1 1.5B — step-by-step reasoning

p — Pi Coding Agent (reads/edits/runs files on your system)

# Interactive session (Groq Llama 70B by default)
p                                    # start in current dir
cd ~/craftededgesolutions.africa && p   # coding session on your site

# One-shot
pask "add a WhatsApp floating button to index.html"
pask "fix the mobile nav menu in style.css"
pask "write a Python script to batch-rename files in ~/content/"

# Model variants
p70b     # Llama 3.3 70B (default, best)
pfast    # Llama 3.1 8B (faster, lighter tasks)
pcode    # Qwen3 32B (code-focused)
pscout   # Llama 4 Scout (newest)
p-or     # OpenRouter backup (when Groq is busy)

# Pipe-in shortcuts
cat buggy.js | fixme
cat code.py  | explain
cat code.js  | review

Pi reads/edits actual files on your machine. Use cd into your project first.

h — Hermes Agent (Spotify, Web Search, Scheduling)

# Interactive

# One-shot
hask "search for top web design trends in Kenya 2026"
hask "create a cron job that emails me a weekly content idea every Monday 9am"

# Spotify

# Model options
h70b          # Llama 3.3 70B via Groq (default)
hfast-groq    # Llama 3.1 8B via Groq (lighter)
hthink        # DeepSeek R1 70B via Groq (reasoning)
hgemma        # Gemma 4 31B via OpenRouter (backup)
hlocal        # gemma3 4B local (offline/private)

Hermes needs 64K+ context — local 3B models won't work as Hermes providers.

Troubleshooting

Problem Fix
groq returns nothing echo $GROQ_API_KEY — if empty, open new terminal
Spotify 401 spotify-refresh
No Spotify device Open Spotify app on phone first
OpenWebUI blank docker restart open-webui
Ollama not responding sudo systemctl restart ollama
7B model crashes Expected — use qwen7 (CPU) or groq (cloud)
Listmonk down docker start listmonk_app
Website 8000 down cd ~/craftededgesolutions.africa && python3 -m http.server 8000 &
Pi TPM error (413) Pipe targeted content instead of letting Pi read full files
Hermes TPM error Its system prompt is large — Groq free tier handles normal use fine
Keys not loading Put them at TOP of ~/.bashrc before case $- block ✓ already done
